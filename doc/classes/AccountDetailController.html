\AccountDetailController

AccountDetail controller, handling all actions that concern a single account.

The following actions are supported:

index action
createentry action
editentry action
createcategory action
inviteuser action

charts action credentials action

indexAction()

indexAction()

Index action shows the basic overview of the current account.

User_id is fetched from session, account_id is a get-parameter. All account entries are fetched from this the database and given to the view.

Tags

None found

createentryAction()

createentryAction()

CreateEntry action shows the create entry form on get.

On post, the values are taken from the form and a new entry object is written into the database. Account_id and user_id is taken from the session.

Tags

None found

editentryAction()

editentryAction()

EditEntry action renders the edit entry from, which is similar to the create entry form, but fills the form elements with the entry data that is fetched from database.

The fetch works with the entry id which is a get-parameter. Account_id is taken from the session.

Tags

None found

createcategoryAction()

createcategoryAction()

CreateCategory action shows the create category form on get.

On post, the name of the new category is taken from the form and written in the database, with the account_id from session.

Tags

None found

inviteuserAction()

inviteuserAction()

InviteUser action shows the invite form on get.

On post, it tries to write the email from the invite form in the user_in_account relationship, with the confirm flag of 0.

Tags

None found

chartsAction()

chartsAction()

Charts action fetches all entries of the account (account_id is get-parameter, user_id is in session), aggregates them and gives them to the view. There, charts are drawn via javascript.

Tags

None found

credentialsAction()

credentialsAction()

Credentials action has an empty body and shows only the corresponding view.

This action is called when another action is called with insufficient credentials (e.g. trying to access an account by manipulating the URL).
